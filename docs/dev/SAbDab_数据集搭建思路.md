
1. 引言：抗体药物开发中的跨物种结构数据需求

在现代生物医药研发领域，单克隆抗体（Monoclonal Antibodies, mAbs）已成为治疗癌症、自身免疫性疾病和感染性疾病的主流手段。然而，抗体药物从实验室走向临床的过程中，面临着一个巨大的转化鸿沟：临床前药效学和毒理学研究通常依赖于啮齿类动物模型（主要是小鼠，Mus musculus），而最终的治疗靶标却是人类（Homo sapiens）抗原。由于物种间的进化差异，人类抗原与其小鼠直系同源物（Orthologs）在氨基酸序列和三维结构上往往存在显著差异。这种差异导致针对人类抗原筛选的高亲和力抗体，往往无法识别小鼠抗原，或者结合亲和力极低，从而无法在常规小鼠模型中评估药物的有效性和安全性。
为了解决这一问题，研究人员通常被迫开发“替代抗体”（Surrogate Antibodies），即专门针对小鼠抗原筛选的、仅用于动物实验的抗体。然而，替代抗体与临床候选抗体在结合表位、亲和力动力学以及效应功能上可能存在本质区别，导致动物实验数据的预测价值大打折扣。理想的解决方案是开发具有“跨物种反应性”（Cross-reactivity）的抗体，即能够同时识别及结合人类和小鼠抗原的分子。
本报告旨在构建一个专门服务于跨物种抗体工程的高质量结构数据集。该数据集的核心单元被定义为一个结构三元组：<鼠源抗原> <人源抗原> <抗体模版>。通过构建这样的数据集，计算生物学家和抗体工程师可以深入分析同一抗体骨架（Antibody Template）在面对两个不同物种的抗原时，其结合界面的结构适应性、构象调整以及相互作用能的差异。这不仅为亲和力成熟（Affinity Maturation）和人源化（Humanization）提供了直接的结构指导，更为基于人工智能（AI）的从头抗体设计提供了宝贵的训练数据。
本报告将基于牛津大学统计系开发的结构抗体数据库（Structural Antibody Database, SAbDab）(1)，详尽阐述如何从海量的蛋白质结构数据中挖掘、筛选、处理并生成符合上述要求的三元组数据文件。我们将深入探讨数据库的架构、生物信息学映射策略、文件格式转换的技术细节，并通过对血清白蛋白、CTLA-4、PD-1等关键靶点的深度案例分析，展示该数据集的科学价值与应用潜力。

2. SAbDab数据库架构与数据挖掘策略

SAbDab 是目前公认的最全面、注释最详尽的抗体结构资源库。与原始的蛋白质数据库（PDB）相比，SAbDab 的核心优势在于它对抗体结构进行了标准化的处理和注释，特别是对抗体可变区（Fv）的编号（如Chothia, IMGT）和配对信息的整理 (1)。要从 SAbDab 中提取符合需求的跨物种三元组，必须首先深入理解其数据组织形式和检索逻辑。
2.1 SAbDab 摘要文件（Summary File）的深度解析

SAbDab 提供了一个核心的摘要文件（summary file），通常以制表符分隔的 .tsv 格式存在。该文件是进行大规模数据挖掘的地图。为了构建我们的数据集，必须对文件中的关键字段进行精确的解析和过滤 (1)。

2.2 数据挖掘的逻辑挑战与解决方案

在构建 <鼠源抗原> <人源抗原> <抗体模版> 数据集时，最直接的挑战在于：PDB中很少有单个条目同时包含这三者。绝大多数情况下，数据是分散的。我们需要处理以下几种数据分布场景：
- 场景 A（理想但稀缺）： 同一研究团队解析了同一抗体分别与人源抗原和鼠源抗原结合的复合物结构。例如，抗体 Fab 片段分别与人血清白蛋白和鼠血清白蛋白共结晶。这种情况下，我们可以直接获得完美匹配的三元组。
- 场景 B（最常见）： SAbDab 中存在抗体与人源抗原的复合物结构（PDB_1）。鼠源抗原的结构存在于 PDB 中，但它是作为一个独立的条目（PDB_2），可能是未结合状态（Apo），也可能是结合了另一个完全不同的抗体。
- 场景 C（治疗性抗体）： 在 Thera-SAbDab 中，许多药物（如 Ipilimumab）只有与人源抗原的复合物。关于其与鼠源抗原不结合或结合较弱的信息存在于文献中，而鼠源抗原本身的结构可能通过替代抗体（Surrogate Antibody）的复合物结构被解析。
针对上述场景，我们的数据构建策略必须是**“以抗体-人抗原复合物为核心，虚拟配对鼠源抗原”**。具体步骤如下：
1. 锚定人源复合物： 在 SAbDab 中筛选出所有 antigen_species 为 Homo sapiens 且 antigen_type 为 protein 的条目。这些条目提供了 <人源抗原> 和 <抗体模版>。
2. 抗原身份识别： 利用 PDB ID 和链 ID，通过 SIFTS 数据库或 UniProt 映射服务，确定人源抗原的 UniProt ID（例如 P02768 代表人血清白蛋白）。
3. 直系同源映射（Ortholog Mapping）： 利用 UniProt API，查找该人源蛋白对应的 Mus musculus 直系同源蛋白的 UniProt ID（例如 P07724 代表小鼠血清白蛋白）。在此步骤需引入序列一致性（Sequence Identity）过滤器，剔除同源性过低（如 < 50%）的配对，因为过低的同源性意味着结构折叠可能完全不同，不具备结构比对的意义。
4. 检索鼠源结构： 使用鼠源 UniProt ID 反向检索 PDB 或 SAbDab，寻找包含该鼠源蛋白的结构。
  - 优先级 1：与同一抗体结合的结构。
  - 优先级 2：与不同抗体结合的结构（全构象抗原）。
  - 优先级 3：未结合的 Apo 结构。
5. 虚拟三元组构建： 将检索到的鼠源抗原结构下载，并透过结构比对算法（如 PyMOL align 或 Biopython Superimposer）将其叠合（Superimpose）到人源复合物中的抗原位置。这样生成的 <鼠源抗原> 文件，实际上是“如果抗体以相同模式结合，鼠源抗原应该所处的空间位置”。这对于计算抗体与鼠源抗原之间的位阻（Clash）或接触势能（Contact Potential）至关重要。

2.3 Thera-SAbDab 的特殊价值

Thera-SAbDab 是 SAbDab 的一个子集，专门收录治疗性抗体 (1)。这些抗体通常拥有更丰富的文献数据，特别是关于种属交叉反应性的实验数据（如 SPR 或 ELISA 测定的 Kd 值）。在构建数据集时，应优先标记来自 Thera-SAbDab 的条目。因为这些抗体经过了深入的工程化改造，其无法结合鼠源抗原往往是设计上的痛点，或者是通过转基因小鼠（如 HuMAb-Mouse）筛选出来的全人源抗体。分析这些数据有助于理解“为了获得高亲和力的人源结合，牺牲了哪些对鼠源抗原的结合能力” (4)。

3. 生物信息学流水线：从UniProt映射到PDB提取

为了实现上述策略，必须建立一套自动化的生物信息学流水线。本节将详细描述技术实现的路径，包括数据库接口的调用、链的识别与清洗。
3.1 核心数据源的链接与映射

SAbDab 的 summary file 虽然提供了链信息，但并未直接提供 UniProt ID。因此，第一步是建立 PDB 链与 UniProt ID 的桥梁。
- SIFTS (Structure Integration with Function, Taxonomy and Sequence)：这是连接 PDB 和 UniProt 的金标准资源。通过解析 SIFTS 提供的 XML 或 CSV 映射文件，可以将 5FUO_A（PDB ID _ Chain ID）精确映射到 P02768（UniProt ID）。
- UniProt API 的使用：
  - 查询直系同源物：获得人源 ID（如 P02768）后，通过 UniProt 的 REST API 查询该蛋白的 gene name（如 ALB），并限定 organism ID 为 10090（小鼠）。
  - 序列比对验证：获取人源和鼠源的 FASTA 序列，进行全局比对（Global Alignment）。如果序列长度差异过大或关键结构域缺失，应在此时标记并剔除。

3.2 晶体结构中的“陷阱”与清洗

在提取结构文件时，必须处理 PDB 数据的天然缺陷：
1. 缺失残基（Missing Residues）：晶体结构中，处于高度动态的环区（Loop）往往电子密度缺失。如果抗原上的缺失区域恰好位于抗原-抗体结合界面（Epitope），那么该结构数据对于训练是无效甚至有害的。流水线必须检查 REMARK 465 记录，计算界面残基的缺失率。如果界面缺失超过 5%，该三元组应被标记为低质量或剔除。
2. 异质原子（HETATM）：PDB 文件包含大量水分子、离子和结晶试剂（如甘油、硫酸根）。
  - 对于抗体模板：通常应剔除所有 HETATM，只保留蛋白质链。
  - 对于抗原：需谨慎处理。某些抗原的结构完整性依赖于金属离子（如钙离子在某些 C 型凝集素或钙粘蛋白中）。如果离子位于结合界面，剔除它会导致静电势计算错误。建议保留紧密结合的金属离子，剔除溶剂水分子。
3. 多聚体状态（Biological Assembly）：抗原在晶体中可能是多聚体（如二聚体 VEGF）。SAbDab 通常基于非对称单元（Asymmetric Unit）提供数据。如果抗体结合的是二聚体界面，仅提取单链抗原是不够的。但根据用户需求形式 <鼠源抗原> <人源抗原>，通常暗示单链对单链的比对。为了简化，我们优先选择抗体 Fv 区与抗原单体 1:1 结合的复合物，或者明确处理多链抗原为一个整体文件。

3.3 复杂性处理：多拷贝与冗余

一个 PDB 文件可能包含多个复合物拷贝（例如非对称单元中有 2 个 Fab 和 2 个抗原）。
- 去冗余策略：对于同一 PDB ID，计算所有拷贝的结构差异。通常它们是非常相似的。选择 B 因子（温度因子）最低、且电子密度最完整的那一套拷贝作为代表。
- 链 ID 的标准化：原始 PDB 的链 ID 可能是 A, B, C, D... 为了数据集的统一性，建议在生成最终文件时将抗体重链重命名为 H，轻链重命名为 L，抗原重命名为 A（或其他固定标识），并在文件名或元数据中记录原始 ID。

4. 结构数据处理技术规范：PDB/CIF文件分割与配准

用户明确要求输出形式为 3 个 cif 或 pdb 文件。这一节将提供具体的操作规范，指导如何生成这些文件。

4.1 文件格式的选择：PDB vs. mmCIF

虽然用户允许使用 PDB 或 CIF，但在技术层面上，mmCIF (macromolecular Crystallographic Information File) 是更优的选择，也是目前 PDB 数据库的归档标准 (6)。
- PDB 格式的局限：传统的 .pdb 格式受限于 80 字符的列宽，且链 ID 只有一位字符，原子序列号有上限。对于大型复合物，PDB 格式可能无法完整描述。
- mmCIF 的优势：无大小限制，元数据更丰富。
- 实施建议：流水线应优先下载 .cif 文件进行解析。但考虑到许多下游对接（Docking）软件或机器学习模型仍偏好 .pdb 格式，建议在分割和处理后，将单链或较小的复合物保存为 .pdb 格式，以提高兼容性。同时分割出来后，也保留.cif文件以便相关的软件或者模型的需要。

4.2 文件生成步骤详解

假设我们已经锁定了人源复合物 5FUO（抗体 CA645 + 人血清白蛋白）和鼠源抗原 8OI2（鼠血清白蛋白）。我们需要生成三个文件：

步骤 1：生成 <抗体模版> (Antibody Template)

1. 输入：5FUO.cif
2. 解析：识别抗体链。SAbDab 指出重链为 H，轻链为 L (7)。
3. 操作：
  - 提取链 H 和链 L 的所有原子。
  - 剔除水分子。
  - 关键点：保持 H 和 L 之间的相对空间位置不变。这代表了抗体结合状态下的构象（Bound Conformation）。
  - 编号标准化：利用 SAbDab 提供的 Chothia 或 IMGT 编号方案重新编号残基，这对后续的序列比对至关重要。
4. 输出：保存为 5FUO_antibody.pdb和5FUO_antibody.cif。

步骤 2：生成 <人源抗原> (Human Antigen)

1. 输入：5FUO.cif
2. 解析：识别抗原链。SAbDab 指出抗原链为 A。
3. 操作：
  - 提取链 A 的所有原子。
  - 关键点：保持其坐标系与原始 PDB 一致。这意味着它与 5FUO_antibody.pdb 是天然对齐的，直接打开两个文件即可看到复合物结构。
4. 输出：保存为 5FUO_human_antigen.pdb 和 5FUO_human_antigen.cif。

步骤 3：生成 <鼠源抗原> (Mouse Antigen)

这是最复杂的一步，涉及结构配准。
1. 输入：8OI2.cif（鼠血清白蛋白）。
2. 解析：提取鼠源抗原链（假设为链 A）。
3. 结构比对（Structural Alignment）：
  - 目标：将鼠源抗原“移动”到人源抗原的空间位置，使其模拟与抗体结合的状态。
  - 算法：使用基于序列比对的结构叠合算法。首先将鼠源序列与人源序列比对，找到保守的 Cα 原子对。然后计算旋转平移矩阵（Rotation-Translation Matrix），最小化这些 Cα 原子对的 RMSD（均方根偏差）。
  - 工具：可以使用 PyMOL 的 align 命令（基于序列预比对）或 super 命令（基于结构特征），或者 Biopython 的 Bio.PDB.Superimposer 模块。
  - 操作：应用计算出的矩阵变换鼠源抗原的所有原子坐标。
4. 输出：保存变换后的坐标为 5FUO_mouse_antigen_aligned.pdb和 5FUO_mouse_antigen_aligned.cif。

4.3 最终数据集的形态

对于每一个数据点（Data Point ID: DP_001），文件夹中将包含：
1. DP_001_mouse_ag.pdb以及 cif 文件: 鼠源抗原结构（已叠合到人源位置）。
2. DP_001_human_ag.pdb:以及 cif 文件 人源抗原结构（原始结合位置）。
3. DP_001_antibody.pdb以及 cif 文件: 抗体 Fv 或 Fab 结构。
这种数据形态允许研究人员直接计算：
- RMSD: mouse_ag 与 human_ag 的结构差异。
- Steric Clashes: mouse_ag 与 antibody 之间的原子碰撞（如果是严重碰撞，说明抗体无法直接结合鼠源抗原）。
- Interface Energy: 计算 human_ag + antibody 的结合能 vs mouse_ag + antibody 的结合能。

5. 深度案例分析 I：血清白蛋白结合抗体的跨物种机制 (CA645 & NbAlb1)

为了验证上述策略的有效性，我们深入分析血清白蛋白（Serum Albumin, SA）系统的结构数据。这是一个极佳的“黄金标准”案例，因为存在丰富的高分辨率晶体结构，涵盖了人源和鼠源抗原与不同抗体的结合状态。
5.1 CA645抗体：半衰期延长的工程化典范

CA645 是一种经过高度工程化的人源化 Fab 片段，其设计目的是通过结合血清白蛋白来延长药物的血清半衰期 (8)。
- 人源复合物结构（PDB: 5FUO）：解析了 CA645 与人血清白蛋白（HSA）的复合物。结构显示 CA645 结合在 HSA 的结构域 II（Domain II）上。
- 跨物种反应性：文献表明 CA645 对人类、食蟹猴和小鼠的血清白蛋白具有相似的纳摩尔级亲和力（1-7 nM）。这一数据点非常关键，意味着 5FUO 中的抗体模板不仅适应人源抗原，也适应鼠源抗原。
- 结构差异分析：通过构建我们的三元组，将鼠源白蛋白（如 PDB 8OI2 链 A）叠合到 5FUO 的 HSA 上，我们可以观察到结合界面的保守性。虽然 HSA 和 MSA（鼠白蛋白）的总体序列一致性约为 72-76% (9)，但在 CA645 结合的特定表位区域，关键残基高度保守。这种“结构保守性”是 CA645 能够跨物种工作的物理基础。

5.2 NbAlb1纳米抗体：独特的结合模式

另一个重要的结合分子是纳米抗体 NbAlb1（PDB: 8OI2）。
- 鼠源复合物结构（PDB: 8OI2）：解析了 NbAlb1 与人血清白蛋白的复合物，但研究同时深入探讨了其与鼠源白蛋白的结合 (10)。
- 关键位点 A364 vs D364：结构分析揭示，NbAlb1 识别位于结构域 II 的一个外周表位。在这个表位中，残基 364 起到了关键作用。在人血清白蛋白（HSA）中它是丙氨酸（A364），而在鼠血清白蛋白（MSA）中它也是中性/疏水残基，但在牛血清白蛋白（BSA）中变成了带负电的天冬氨酸（D364）。这解释了为什么 NbAlb1 能结合人和鼠，但不能结合牛白蛋白 (11)。
- 数据集价值：通过比较 8OI2（NbAlb1+HSA）和虚拟配对的 NbAlb1+MSA，我们的数据集可以清晰地展示 A364 这一位点在两种抗原中的空间环境，训练算法识别这种“允许的突变”与“破坏性的突变”（如 A -> D）。

6. 深度案例分析 II：免疫检查点抑制剂的结构差异 (CTLA-4 & PD-1)

免疫检查点是抗体药物研发的热点，但也是跨物种反应性问题最突出的领域。这里的结构数据揭示了为什么很多临床抗体在小鼠模型中无效。
6.1 CTLA-4：Ipilimumab 与替代抗体的结构鸿沟

Ipilimumab 是第一款获批的抗 CTLA-4 全人源抗体。
- 人源复合物（PDB: 3TRU / 6RP8）：Ipilimumab 结合人 CTLA-4 的结构显示，它识别一个主要位于前表面的表位。
- 鼠源抗原的差异：小鼠 CTLA-4 与人 CTLA-4 在 BC 环（BC loop）区域存在显著的构象差异 (13)。Ipilimumab 的 CDR 环无法适应小鼠 CTLA-4 的 BC 环构象，导致严重的立体位阻。
- 替代抗体 9D9（PDB: 9DQ5）：为了在小鼠中进行实验，研究人员使用了替代抗体 9D9。结构对比显示，9D9 结合小鼠 CTLA-4 的方式与 Ipilimumab 结合人 CTLA-4 的方式完全不同 (14)。
- 数据集应用：在我们的数据集中，将 9DQ5 的鼠 CTLA-4 叠合到 6RP8 的人 CTLA-4 上，将直观地显示 Ipilimumab 与鼠抗原的“碰撞”。这为计算设计提供了极佳的“负样本”（Negative Sample），即“什么样的结构差异会导致结合失败”。

6.2 PD-1：Pembrolizumab 的结合界面

Pembrolizumab（Keytruda）是抗 PD-1 的重磅药物。
- 人源复合物（PDB: 5GGS）：揭示了 Pembrolizumab 极其依赖于 PD-1 的 C'D 环 (16)。
- 鼠源抗原（PDB: 3BP5/3BIK）：小鼠 PD-1 的结构虽然在整体折叠上与人 PD-1 相似（IgV 结构域），但在 C'D 环和 FG 环的精细结构及表面电荷分布上存在差异 (18)。
- 糖基化问题：PD-1 是高度糖基化的蛋白。人 PD-1 和鼠 PD-1 的糖基化位点（N-glycosylation sites）不同。PDB 结构通常切除了糖链或只保留了根部（NAG）。在构建数据集时，必须注意这些“缺失”的糖链可能在真实生物体内造成的位阻。虽然 PDB 文件不包含完整糖链，但通过序列比对（N-X-S/T 基序）可以推断出潜在的糖基化差异对跨物种结合的影响 (20)。

7. 深度案例分析 III：经典模型与病毒抗原 (溶菌酶, Influenza, HIV)

除了治疗性靶点，经典的免疫学模型提供了最详尽的亲和力成熟数据。
7.1 溶菌酶（Lysozyme）：HyHEL 系列抗体

这是历史上研究最透彻的抗原-抗体系统之一。
- 抗体 D1.3 和 HyHEL-10：这些抗体针对鸡蛋清溶菌酶（HEL）产生 (22)。
- 人/鼠溶菌酶：PDB 中存在人溶菌酶（1REX）和小鼠溶菌酶 M（1IVM）的高分辨率结构。
- 结构硬度：溶菌酶是一个相对刚性的蛋白，这使得它成为测试“刚体对接”（Rigid Body Docking）算法的理想基准。将 1IVM（鼠）叠合到 1FDL（D1.3-HEL复合物）中的抗原位置，可以精确计算每一个界面残基突变（如 R -> K, L -> M）对结合能的微小贡献。这种精度是柔性较大的 CTLA-4 系统难以比拟的。

7.2 病毒抗原的广谱性与跨物种概念的延伸

虽然病毒不涉及“人 vs 鼠”的物种差异，但“针对不同毒株的广谱中和抗体”在结构逻辑上与跨物种抗体完全一致。
- Influenza Hemagglutinin (HA)：针对 HA 茎部（Stem）的抗体（如 CR6261, F10）通常具有极宽的反应谱。数据集可以包含 <H1N1 HA> <H5N1 HA> <Antibody> 的三元组，用于训练识别保守疏水槽的算法 (24)。
- HIV gp120：V3 环抗体在不同毒株间的亲和力成熟过程，为理解抗体如何通过引入体细胞突变来适应抗原变异提供了动态视角 (26)。

8. 数据缺陷处理与未来展望

在构建完上述数据集后，不可避免地会遇到数据缺失的问题。
8.1 晶体结构的局限与补救

并非所有的鼠源直系同源物都有已解析的晶体结构。
- 同源建模（Homology Modeling）：如果小鼠抗原没有 PDB 结构，但与人源抗原序列一致性很高（>80%），可以使用 MODELLER 以人源结构为模板构建小鼠模型。
- AlphaFold2 预测：对于结构未知的鼠源抗原，AlphaFold2 提供了高可信度的预测模型（如 AF-Q02242 对应小鼠 PD-1）。虽然预测模型的侧链精度不如晶体结构，但在缺乏实验数据的情况下，它们是极佳的替代品。在构建数据集时，应标记该结构来源为“Predicted”，以区分于实验结构 (27)。

8.2 数据的标准化与清洗

生成的 PDB 文件必须经过严格的清洗：
- Renumbering：统一使用 IMGT 编号，不仅针对抗体，如果可能，对抗原也进行基于多序列比对的通用编号，这将极大地简化机器学习模型的输入特征提取。
- Energy Minimization：虚拟配对（叠合）后的结构可能存在严重的原子重叠（Clash）。在发布数据集前，建议使用 Rosetta 或 Amber 对界面侧链进行轻度的能量最小化（Energy Minimization），以消除非物理的接触，提高数据质量。

8.3 总结

本报告详细规划了构建 <鼠源抗原> <人源抗原> <抗体模版> 结构数据集的完整路径。通过以 SAbDab 为枢纽，结合 UniProt 的物种映射和 PDB 的结构提取，我们可以生成一个涵盖多种结合模式、亲和力范围和抗原类型的高质量数据集。该数据集不仅包含了血清白蛋白、CTLA-4、PD-1 等热门靶点的实验数据，还通过生物信息学手段填补了跨物种配对的空白。这将成为抗体工程领域，特别是计算机辅助抗体设计（CADD）和人工智能药物发现（AIDD）的重要基础设施。
